Stage 1

Information Stage :

  • Nom de l'entreprise : CRET
  • Nom du projet: Gestionnaire de facturation
  • Secteur d'activité: Opérateurs/Reseau Telecoms
  • Rapport de Stage: Stage CRET

Objectif :

Créer un gestionnaire de facturation capable d'automatiser la création et le renouvellement des factures selon une périodicité définie.

Mes missions :

Mission 1 : Création de la base de données et connexion

  • - Conception de la structure BDD
  • - Connexion avec PDO
  • - Insertion de données de test

Mission 2 : Implémentation des fonctionnalités CRUD

  • - Création des formulaires d'ajout
  • - Affichage dynamique des factures
  • - Mise à jour et suppression des clients

Mission 3 : Automatisation du renouvellement des factures

  • - Ajout du bouton "Facturer"
  • - Mise à jour automatique des dates
  • - Gestion du renouvellement périodique

Outils Utilisés :

PHP

PHP m'a servi à gérer la logique de l'application côté back-end.

HTML/CSS

HTML et CSS ont permis de structurer et de styliser l'interface utilisateur côté front-end.

MYSQL

J'ai utilisé MySQL pour stocker et gérer les données de l'application.

VISUAL STUDIO CODE

J'ai utilisé Visual Studio Code pour écrire et organiser mon code de façon efficace. Il m'a permis d'avoir une interface fluide tout en bénéficiant de fonctionnalités avancées.

DOCKER

Docker m'a aidé à lancer phpMyAdmin pour gérer la base de données sans configuration complexe.

GITHUB

Avec GitHub, j'ai sauvegardé et partagé les versions du projet facilement.

Problèmes Rencontrés :

1. Problème de mise à jour des dates :
  • Vérification des variables récupérées
  • Vérification du format des dates
2. Problèmes liés à la base de données :
  • Vérification de la connexion à la base de données
  • Vérification de la syntaxe des requêtes SQL
3. Erreurs de syntaxe dans le code :
  • Vérification des parenthèses et des points-virgules
  • Vérification de la structure des blocs PHP et HTML

Conclusion :

Ce projet m'a permis de renforcer mes compétences en gestion de base de données, développement PHP avec PDO, et manipulation des dates. J'ai appris à appliquer la logique CRUD, à valider et assainir les données, et à gérer les erreurs pour créer des applications robustes.

Pour mon avenir en tant que développeur progiciel, ce projet m’a permis de comprendre les exigences des applications métier, d’automatiser des processus et de gérer des données. Ces compétences sont essentielles pour le développement de logiciels professionnels.
